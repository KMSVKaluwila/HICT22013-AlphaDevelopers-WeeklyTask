# AlphaDevelopers - Semantic HTML5 Webpage Skeleton

This repository contains the foundation for the **AlphaDevelopers** project, developed as part of **Task 01** for the **HICT 22013: Web Design & Development** course.

---

## 1. Project Goal
The primary objective of this website is to introduce our software development team (**AlphaDevelopers**), showcase the core services and features we offer, and provide a way for potential clients to get in touch with us.

---

## 2. Main Sections of the Webpage

* **Header & Navigation:** Includes the website name ("AlphaDevelopers") and a navigation menu (Home, Features, Our Team, Contact) allowing users to easily navigate to different sections of the page.
* **Hero Section:** The first section visible to a user visiting the site. It contains our main tagline ("Empowering Teams to Build Better Software"), a brief introductory description, an illustrative team image, and a Call-to-Action (CTA) button ("Get Started Now").
* **Features Section:** Explains the 3 core values we utilize in our development workflow:
  * **Semantic HTML5:** Using native semantic tags for structure.
  * **Accessibility (a11y):** Structuring content to make it usable for everyone.
  * **Clean Collaboration:** Dividing code cleanly to allow smooth teamwork.
* **Meet Our Development Team:** Introduces our 5 team members along with their specific roles and responsibilities. This section also helps us divide the development tasks effectively among the team.
* **Contact Form & Footer:** Contains a contact form allowing users to submit their name, email address, and message to reach us. The footer holds copyright information and contact details (email, address, and GitHub repository links).

---

## 3. Key Highlights of this Phase (Step 01 Spec)
The `index.html` document is built strictly according to HTML5 standards:
* **Zero Div Soup:** It does not use any `<div>` tags. Layout structuring is handled entirely by native semantic elements.
* **100% Free of CSS/Styles:** There is no external stylesheet, `<style>` block, or inline styling. This structural separation is a strict requirement for this first task, and CSS will be integrated in subsequent weeks.
